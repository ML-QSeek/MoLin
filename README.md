# Molin

> **人机协作 · AI 主导 · 原生业务架构**


## 这是什么

**Molin 是一套面向未来的业务架构方案。**

它重新思考了一个基本问题：**当 AI 具备自主推理和行动能力时，一个组织的业务系统应该长什么样？**

答案不是"在旧系统上加 AI 接口"，而是以 **AI 为第一行动主体** 重新设计整个架构。人是系统的设计者和约束设定者，AI 是业务的一线执行者。


## 核心思想

| 原则 | 含义 |
|------|------|
| **AI 主导 · 人协作** | AI 自主完成业务闭环，人在边界处设定约束、校准异常 |
| **约束即边界** | 系统在约束围成的可行域内自主运行，不越界、不失控 |


## 全景架构

Molin 由十三个独立模块构成，按职能分为四层：

**基础设施层**
QBase 数据底座 · QLens 算力集群 · QField 统一网关

**AI 能力层**
QMulti 认知底座 · QSkills 技能库 · QForge 工程治理 · QSeek 自主推理

**业务层**
QFuse 知识融合 · QSense 态势感知 · QCore 决策中枢 · QFlow 资源调度

**执行层**
ModelLive 仿真验证 · ModelLive Real 执行引擎

每一层独立演进，通过标准化接口协作。模型可以换、算法可以迭代，**架构不动**。
